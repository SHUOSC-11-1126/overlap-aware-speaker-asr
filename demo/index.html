Readable pipeline

The system flow in six readable steps.

Use this slide when the architecture figure is too small on a projector. It is the same story, rewritten as large text.

1. Audio assets
Mixed recordings, separated tracks, and reference transcripts are organized as evidence inputs.
2. ASR routes
Run or compare mixed, separated, and cleaned transcript routes.
3. Metrics
CER, speaker-aware CER, cpCER-lite, and error-type analysis expose different failures.
4. Router
router_v2 selects a route from deployment-visible instability signals.
5. Audits
Synthetic, held-out, source-disjoint, runtime, and risk checks prevent overclaiming.
6. Frontiers
MeetEval, speaker profile, LLM critic, demo excellence, and AudioDepth become scoped follow-ups.

Why fixed separation fails

Separation is a tool, not a law.

Light and mid overlap can suffer from insertion and repetition artifacts after separation. Heavy and opposite overlap benefit more clearly. The router contribution is making that boundary explicit.
